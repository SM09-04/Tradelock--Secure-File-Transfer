Tamper Attack

Flip a single byte in the encrypted file to simulate an adversary modifying data in transit.

What happens: A byte in the ciphertext is XOR'd with 0xFF, corrupting the file. AES-GCM's authentication tag will fail during decryption, immediately detecting the tampering.
🔓

OTP Brute Force

Attempt to guess the 6-digit TOTP code by trying many random values rapidly.

What happens: The server rate-limits OTP attempts and blocks the session after 5 failures. With 106 possible codes, brute force is impractical even without rate limiting.
🔁

Replay Attack

Attempt to reuse an OTP code that was captured from a previous session.

What happens: A TOTP code from 90 seconds ago is replayed. Because TOTP codes change every 30 seconds and have a strict time window, expired codes are rejected.

Defense Summary

🔒
AES-256-GCM

Authenticated encryption provides both confidentiality and integrity. Any modification to ciphertext is detected.

📱
TOTP 2FA

Time-based codes change every 30 seconds. Stolen passwords alone cannot grant access.

🛡️
Rate Limiting

OTP attempts are capped per session, making brute force statistically infeasible.

⏱️
Token Expiry

TOTP codes are valid for 30 seconds with a ±1 window. Replay attacks with old tokens always fail.
